**Linked log entry:** [[log#2026-05-03-pivot-to-ma50w10]]

## Context

Six convergent null results have stacked up against single-class Gate.io spot microstructure:

1. 16h initial dataset
2. 23h dataset
3. 86.7h dataset
4. Bayesian sweep (550 trials, 0 winners)
5. D6 binding 133.7h (🛑 NO-GO, 0 maker-fee winners)
6. **B.2.1 ensemble 202.7h walk-forward** — gross +1.46 bps, net maker −8.54 bps, win rate 1.9%

Meanwhile `MA_STRATEGY.md` documents that a one-line rule — `hold BTC if daily close > SMA50 AND weekly close > SMA10` — returned **+3,427% chained over 7.5 years** of real BTC data (2018–2026), validated across six independent regime samples vs +217% buy-and-hold over the same period.

The 9-agent legacy engine (current default in `run.py`) returned **−0.5%** over 24 months on the same data window where the MA rule returned +74.9%. The vote-ensemble paper bots running on the VPS (4 pairs × $250 paper-real, ETH/SOL/XRP/AVAX) bled −0.32% → −2.38% over 3 days per [[findings/2026-04-28-paper-pnl-bleeding]].

L99 = 60-day Stage 2 real-money profitable track record per `CHAMPION_MODE.md`. Earliest credible date: October 2026. Today is 2026-05-03 → ~5 months. To hit the 60-day track-record window we need to be in Stage 2 by early August 2026. B.2.3 (on-chain B1) is 5+ weeks data cycle + validation + deployment ⇒ Stage 2 unlikely before mid-July, AND with prior probability further reduced after 6 convergent nulls.

## Choice

**Add a paper-real `ma50w10` bot on BTC_USDT 1d as a parallel L99 candidate alongside the existing 4 vote bots.** Don't shut down the vote bots — they remain as control. Re-evaluate at 30-day mark.

Concrete deliverables (this commit):

- `systemd/trading-bot-ma50w10.service` — paper-real bot unit (BTC_USDT, 1d, $250 balance) — INSTALLED ON VPS pending operator
- This decision page
- `wiki/log.md` + `wiki/index.md` updates

Concrete deliverables (operator, on VPS, ~3 minutes):

- Smoke test: 5-loop run to verify `ma50w10` strategy executes against live Gate.io daily candles
- If smoke test passes: install + start `trading-bot-ma50w10.service`
- Verify journalctl shows daily strategy decisions (📈 MA-BUY / ✅ MA-SELL events)

## Why NOT the other branches

| Alternative | Rejected because |
|---|---|
| A. B.2.3 on-chain B1 (per playbook fallback) | 5+ week data cycle + validation + deployment; timeline puts Stage 2 entry past August deadline; prior reduced after 6 nulls. **Defer, not abandon** — re-open if MA paper-real fails the 30-day gate. |
| B. Try other ensemble cluster on B.2.1 | The 86% trade-saturation diagnostic is structural, not cluster-specific. Almost certainly produces a 7th null. Cheap but low EV. |
| C. **Pivot to MA50+W10 paper-real, defer B.2.3** ← chosen | Highest-EV move per project's own validated research. Doesn't burn the on-chain optionality; it just sequences it after the better-prior candidate. |
| D. Switch the 4 vote bots TO ma50w10 | Loses the parallel A/B comparison. Vote bot is the "current SHIP candidate" per `run.py --help`; deserves to keep running as control even if we don't think it'll win. |
| E. Stage 2 immediately with real money | Premature. CLAUDE.md hard rule: 30-day paper-real validation precedes real-money Stage 2 (per ADR pipeline). |

## Consequences

**Enabled:**
- Stage 2 entry on schedule (Aug 2026) IF ma50w10 paper-real shows green at 30-day mark (Jun 2026)
- Parallel A/B between vote-ensemble (current bots) and ma50w10 — direct empirical comparison, not just backtest
- Path back to B.2.3 preserved — `microstructure-collector.service` continues running, parquet keeps accumulating, future re-test cheap

**Costs:**
- One additional systemd service on VPS (~256MB memory cap, 1 daily candle fetch per loop — negligible CPU)
- Adds `systemd/trading-bot-ma50w10.service` (38 lines) to the repo
- 30-day calendar wait before strategy decision

## Reversibility

100%. To remove paper bot:
```bash
sudo systemctl disable --now trading-bot-ma50w10
sudo rm /etc/systemd/system/trading-bot-ma50w10.service
sudo systemctl daemon-reload
git rm systemd/trading-bot-ma50w10.service
```
