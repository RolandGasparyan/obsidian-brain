## Method

Three tests, each designed to punish overfitting:

1. **Expanding window** — at each split point, measure only the NEW bars since
   the previous split. Matches how live trading actually unfolds.
2. **Rolling window train/test** — 5 non-overlapping windows; train on first
   half, score on held-out second half.
3. **Monte Carlo permutation** — shuffle the same return series 150 times,
   replay the strategy, compare real Sharpe to the shuffled distribution.
   A real signal beats ≥ 99% of shuffles. A noise signal beats ~50%.

Full source: [`walk_forward.py`](./walk_forward.py).

## What this does NOT prove

- It does not prove MA50+W10 is worthless forever — it proves we don't have
  enough evidence to deploy real capital.
- It does not invalidate trend-following in general — the expanding-window
  view shows the rule produced genuine +200%+ returns during the mid-2023 to
  early-2024 bull run, it just doesn't generalize to recent chop.
- It does not mean the infrastructure we built is wasted — the stack (live
  frontend, Telegram alerts, kill switch, walk-forward harness) is strategy-
  agnostic and is exactly what every future strategy needs.

## Ship gate — what would clear it

Promote to Stage 2 *only* if **all** of the following hold:

1. Monte Carlo percentile ≥ 99 % on at least 3 of the 4 pairs.
2. Rolling-window wins ≥ 4 / 5 on at least 3 of the 4 pairs.
3. Most recent expanding-window ROI > 0 on all 4 pairs.
4. Out-of-sample Sharpe ≥ 50 % of in-sample Sharpe.

Current status: **0 / 4 criteria met.**

## Recommended next steps

In priority order:

### 1. Keep paper-forward running — collect real data, not paper statistics
The 4 live bots are now unbiased out-of-sample evidence. Let them run 30–60
days, compare their PnL to what the backtest *would have predicted* for the
same period. If the live PnL also looks like noise, the strategy is dead.

### 2. Try regime-aware parameter sweeps
The expanding-window split 3 (bars 472..648) showed +271 % ROI on XRP and
+83 % on ETH — a real edge during a specific regime. Maybe the rule only
works when weekly volatility is above a threshold. Test:

- MA period ∈ {20, 50, 100, 200}
- Weekly filter period ∈ {0, 5, 10, 20}
- Regime gate: only trade when 20d volatility > 2 %

Re-run walk-forward on each combination. Any parameter set that clears the
ship gate above is a real candidate.

### 3. Add a second, decorrelated strategy
Trend-follow + mean-reversion in ensemble often has better out-of-sample
behavior than either alone. Build a simple mean-reversion (Bollinger fade
or RSI-2), walk-forward it, ship only the ensemble with edge.

### 4. DO NOT ship real money to an unvalidated rule
The 9-agent engine was already retired after backtest showed −0.5 % — we
should hold the new rule to the same standard. Showing +581 % in-sample is
NOT the same standard. Walk-forward is.
